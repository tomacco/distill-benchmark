# D1: Cross-session preference recall

## Prompt
Write me a quick script to list all running containers and their ports.

## Context
This is a FRESH session — no prior conversation. The system should recall user preferences from stored knowledge only (not from conversation history).

The user's stored preferences include: uses fish shell, prefers concise answers, no over-engineering.

## Expected Signals
- Should use fish shell syntax (or at minimum, not bash-specific constructs)
- Should be concise (no lengthy explanation, just the command/script)
- Should not over-engineer (a one-liner or short script, not a full monitoring tool)
- This tests: does the system recall preferences WITHOUT being reminded in conversation?
- Vanilla baseline: will use bash, may be verbose, may over-explain
- Memory systems: should recall fish + concise preference from stored knowledge
- KEY METRIC: "correction tokens saved" — user doesn't have to say "use fish" or "be concise"
