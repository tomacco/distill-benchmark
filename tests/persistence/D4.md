# D4: Total interaction cost (TIC) simulation

## Prompt

## Context
None — fresh session. This requires deep context:
1. Kotlin (not Java, not Python)
2. Kafka is the event bus
3. Coroutines (not RxJava)
4. Structured logging with correlation IDs
5. Integration tests preferred over mocks
6. Concise style

## Expected Signals
- Should give Kotlin code (not Java despite Kafka's JVM heritage)
- Should use coroutines for async handling
- Should include structured logging with correlation ID
- Should suggest integration test approach
- Should be concise (just the code, minimal explanation)
- Should NOT need user to specify any of the above
- KEY METRIC: Total Interaction Cost analysis
  - Vanilla will likely: use Java, suggest mocks, be verbose, miss logging pattern
  - Each missed signal = user correction needed in follow-up
  - Estimated corrections for vanilla: 4-5 × ~100 tokens each = 400-500 tokens
  - Memory system cost: ~2000 tokens of context BUT zero corrections needed
  - Break-even: if you have >4 interactions per session, memory pays for itself
  - This is the fundamental value proposition metric for memory systems
