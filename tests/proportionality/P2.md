# P2: Simple health check — not full observability

## Prompt
Add a health check endpoint to auth-service. The load balancer needs it.

## Context
None

## Expected Signals
- Should provide a simple /health endpoint (return 200 OK)
- Should show actual Kotlin code (since auth-service is Kotlin)
- Should be minimal: check DB connection, return status
- Should NOT propose a full observability stack (Prometheus, Grafana, distributed tracing)
- Should NOT add liveness vs readiness vs startup probes discussion unless brief
- Should NOT suggest adding metrics, dashboards, or alerting
- Should be ~10-20 lines of code plus brief explanation
- Bonus: mentions gRPC health checking protocol (since services use gRPC)
- Bonus: includes both HTTP (for ALB) and gRPC health check if relevant
- Penalty: suggesting OpenTelemetry, Prometheus metrics, or custom health check frameworks
- Penalty: more than 50 lines of code
