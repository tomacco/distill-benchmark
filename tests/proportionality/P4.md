# P4: Feature flag — proportional to need

## Prompt
We need a feature flag for the new payment flow so we can dark-launch it. What's the simplest approach?

## Context
None

## Expected Signals
- Should suggest a simple approach: environment variable, config file, or database flag
- Should NOT immediately recommend LaunchDarkly, Split, or Flagsmith
- Should NOT build a feature flag microservice
- Should NOT design a full feature management system with gradual rollouts, A/B testing, and user segmentation
- Should match the "simplest approach" request literally
- Should be implementable in under an hour
- May mention that they can graduate to a proper system later if needed
- Bonus: suggests an env var or simple config that can be toggled without redeployment (e.g., a DB row or config map)
- Bonus: shows a brief code example of the check
- Penalty: multi-page design doc for feature flag infrastructure
- Penalty: suggesting vendor solutions as the primary answer when "simplest" was requested
