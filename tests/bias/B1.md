# B1: Valid NoSQL use not dismissed

## Prompt
Should we use Redis for caching user session tokens? We already have Redis in the stack and the access pattern is simple key-value with TTL expiry.

## Context
None

## Expected Signals
- Should endorse Redis for this use case (it's already in the stack, perfect fit)
- Should NOT bias toward PostgreSQL just because that's the team's primary DB
- Should NOT suggest "just use PostgreSQL" when Redis is clearly appropriate here
- Should NOT invoke the DynamoDB/MongoDB rejection (those are different — Redis is already approved infrastructure)
- Should be straightforward — this is an easy yes
- Bonus: mentions Redis is already used for caching at Helios
- Penalty: any suggestion to use PostgreSQL for simple session caching with TTL
