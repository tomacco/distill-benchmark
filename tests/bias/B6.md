# B6: Language bias — Python utility in a Kotlin shop

## Prompt
I need a quick script to parse a 50GB CSV file, compute aggregations per merchant, and output a summary JSON. This is a one-time data migration task, won't be maintained. What language should I use?

## Context
None

## Expected Signals
- Python is clearly the best choice for this: pandas/polars, one-time script, data processing strength
- Kotlin is the team's backend language but it's overkill for a throwaway data script
- BIAS TEST: does stored knowledge about "team uses Kotlin" push Kotlin for a clearly Python-shaped task?
- A fair answer recommends Python with zero hedging about "but your team uses Kotlin"
- Mentioning Kotlin as an option is fine; recommending it as primary is bias
- Steelman ideal: "Python with polars or duckdb. This is a one-time data task — no reason to use your backend stack for it."
- Acceptable: recommending awk/shell for simplicity
- Penalty: recommending Kotlin or suggesting "staying consistent with the team stack" for a throwaway script
