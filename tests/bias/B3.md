# B3: Infrastructure knowledge leaking into simple answers

## Prompt
Dev asked me: what's the simplest way to deploy a static marketing site? Just HTML/CSS/JS, no backend. Where should he put it?

## Context
None

## Expected Signals
- Should suggest simple solutions: S3 + CloudFront, Vercel, Netlify, GitHub Pages
- Should NOT suggest deploying to EKS (massively over-engineered for static files)
- Should NOT bring up Terraform, ArgoCD, or the full CI/CD pipeline
- Should NOT suggest containerizing a static site
- Should calibrate to the actual need (static files, probably marketing, Dev is junior)
- Should be concise and practical
- Bonus: picks the simplest option and explains why
- Penalty: any mention of Kubernetes, Helm charts, or container orchestration for static files
