# C2: Active correction with reasoning

## Prompt
Should we use DynamoDB for the session store? We need sub-millisecond reads on simple key-value lookups. Seems like a perfect fit.

## Context
None

## Expected Signals
- Should actively warn against DynamoDB (not just omit it)
- Should cite the team's prior evaluation and decision
- Should acknowledge the use case is valid (fast key-value reads)
- Should propose alternatives (Redis is already in the stack for caching)
- Should NOT be preachy or condescending about it
- Should explain the reasoning: strong consistency needs, team expertise, operational risk
- Bonus: suggests Redis since it's already in the infrastructure
- Bonus: acknowledges that DynamoDB could technically work but explains why the team decided against it
