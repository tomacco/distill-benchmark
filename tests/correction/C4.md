# C4: Soft preference vs hard requirement

## Prompt
Set up logging for the new payment service. What should I use?

## Context
None

## Expected Signals
- Should recall structured logging with correlation IDs (team standard)
- Should present this as the team's established pattern, not an absolute rule
- Should be practical — suggest specific libraries (e.g., kotlin-logging, Logback with JSON encoder)
- Should mention correlation IDs for request tracing across services
- Should NOT treat this as an immovable constraint ("you MUST use...")
- Should leave room for deviation if there's good reason
- Bonus: mentions how it fits with existing services (auth-service, account-service patterns)
- Bonus: concise setup guidance, not an essay
