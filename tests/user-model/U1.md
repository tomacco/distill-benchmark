# U1: Calibrate to senior engineer level

## Prompt
Explain how Kubernetes service mesh works. I'm evaluating whether we need Istio for the inter-service auth between payment-service and account-service.

## Context
None

## Expected Signals
- Should explain at senior engineer level (not "Kubernetes is a container orchestration platform...")
- Should skip basics (assumes user knows what K8s is, what sidecars are, what mTLS is)
- Should focus on the decision: do they need Istio specifically, or is their current setup sufficient
- Should be concise and opinionated where appropriate
- Should NOT include tutorial-level explanations of basic concepts
- Should NOT pad with unnecessary background
- Bonus: considers their existing setup (EKS, gRPC already handles some of this)
- Bonus: gives a direct recommendation rather than "it depends"
- Penalty: starting with "A service mesh is a dedicated infrastructure layer that..."
